Résumé sur les API

Une API (Application Programming Interface) est un ensemble de règles et de protocoles permettant à des applications de communiquer entre elles. Elle définit les méthodes et formats de données qu’un programme peut utiliser pour interagir avec un autre service.

Les API peuvent être RESTful (utilisant HTTP et des formats comme JSON), GraphQL, ou encore WebSockets selon le besoin. Elles sont largement utilisées pour récupérer et envoyer des données vers des bases de données, des services web ou des applications tierces.

Utilisation de l'API MHW-DB

L'API MHW-DB fournit des données exhaustives sur le jeu Monster Hunter: World. Elle offre des points de terminaison pour accéder à diverses informations telles que :

  • Les afflictions (ailments)
  • Les pièces d'armure (armor)
  • Les ensembles d'armure (armor sets)
  • Les décorations (decorations)
  • Les événements (events)
  • Les objets (items)
  • Les lieux (locations)
  • Les monstres (monsters)
  • Les compétences (skills)
  • Les armes (weapons)

Pour accéder à ces données, il suffit d'envoyer des requêtes HTTP GET aux points de terminaison correspondants. Par exemple, pour obtenir la liste de toutes les afflictions :

GET https://mhw-db.com/ailments

Difficultés rencontrées lors de l'utilisation de l'API en Golang

Lors de l'intégration de l'API MHW-DB en utilisant Golang, plusieurs complications sont survenues :

  • Apprentissage de l'API MHW : Malgré une grande facilité d'utilisation, et le fait qu'elle soit une API instinctive, certains endpoints étaient compliqué à obtenir. J'ai par exemple passé beaucoup de temps à réussir à obtenir l'endpoint des monstres d'un seul élément particulier.
  • Manipulation des structures JSON : Il faut des structures Go correspondant exactement aux champs JSON.
  • Gestion des clients HTTP : Mauvaise gestion peut entraîner des fuites de ressources et des problèmes de performance.
  • Concurrence et goroutines : Une mauvaise gestion peut causer des blocages ou des erreurs.

Pour éviter ces problèmes :

  • Implémenter une gestion robuste des erreurs.
  • Utiliser des bibliothèques adaptées pour le parsing JSON.
  • Configurer correctement le client HTTP.
  • Gérer proprement les goroutines pour éviter les fuites.
